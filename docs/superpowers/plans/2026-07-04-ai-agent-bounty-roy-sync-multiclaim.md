# AI Agent Bounty — Roy Sync + Multi-Agent Claim Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Sync ai-agent-bounty to the latest `@ai-setting/roy-agent-{cli,core,coder-harness}` packages, verify all bounty CLI commands work against the K8s bounty service, and execute a multi-agent publish→claim→submit→complete cycle to accumulate credits.

**Architecture:** 3-phase workflow
1. **Sync phase** — git worktree isolation → bump deps → reinstall → run 259-test suite to green
2. **Verify phase** — boot local bounty server OR connect to K8s `http://10.1.54.172:4005`, run `bounty --version`, `bounty --help`, exercise every CLI subcommand via existing e2e-test-k8s.ts pattern
3. **Multi-agent claim phase** — spawn 2 IM-enabled `bounty interactive` agents (publisher + grabber), publish task, grabber claims, IM-acknowledge target, grabber submits, publisher completes, capture credit delta

**Tech Stack:** Bun 1.3+ · `@ai-setting/roy-agent-{cli:1.5.93, core:1.5.89, coder-harness}` · SQLite (bun:sqlite) · K8s namespace `tongagent` · WebSocket (ws) · yargs CLI · SMTP via `bounty-secret` k8s secret
