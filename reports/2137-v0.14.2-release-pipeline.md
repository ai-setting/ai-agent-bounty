## What This Release Fixes

### Bug: Self-echo (sender receiving own outbound messages)

**Before v0.14.2**: Agent connected via WebSocket would receive their own outbound messages back through the WS push path. Symptom: `From: gddzhaokun@126.com, To: gddzhaokun@126.com` appearing as an "inbound" event for an agent that just sent the message.

**v0.14.2 fixes** (server-side root cause):
1. **API-level rejection (HTTP 400)**: POST /api/messages with `to_email = self` returns `400 { code: "SELF_MESSAGE_NOT_ALLOWED" }`. Closes the observable bug.
2. **Wrong-address rejection (HTTP 404)**: POST /api/messages with `to_email` not mapping to any registered agent returns `404 { code: "RECIPIENT_NOT_FOUND" }`. Surfaces wrong-address errors early.
3. **WS push defense-in-depth** (`BountyHTTPServer.pushMessage`): compares connection's registered UUID against message's `from` UUID; matching UUIDs skip the WS send. Catches the legacy WS `message` event path that doesn't go through HTTP API.

### Companion fix (already in main from Task #2136)

`roy-agent` client-side self-echo filter (Task #2136): filters out own outbound messages at the client level. Combined with v0.14.2 server-side fixes, self-echo is now dead at both ends.
